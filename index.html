1
Trigger + Get Response + Initialize Variables
Trigger · 3 actions
Trigger When a new response is submitted — select your IR form Action 1 Get response details → Form ID: same form · Response ID: triggerOutputs()?['body/resourceData/responseId'] Action 2 Initialize variable: varIRNumber = concat('IR-', formatDateTime(utcNow(), 'yyyyMMdd-HHmm')) Action 3 Initialize variable: varStatus = Pending Review Action 4 Initialize variable: varAllComments (Array) = [] Action 5 Initialize variable: varSelectedNotes (String) = empty

Send an email (V2) × 1 email, 4 recipients

One email, all four on the To/CC line. Summary view only — no buttons, no action. Subject prefix makes it instantly distinguishable from the action emails.

4
Condition — Check Emily's Response
Condition
Condition outputs('Emily_Approval')?['body/responses'][0]['displayText'] is equal to Return with Notes

YES — Return Branch

1. Set varStatus = Returned by Clinical Director
2. Send email to submitter email from form response:
— Subject: [IR RETURNED] @{varIRNumber} — Clinical Director Review Notes
— Include Emily's comments from varAllComments
— Instruct submitter to resubmit corrected IR
3. Update SharePoint item status = Returned
4. Terminate the flow

NO — Proceed to Compliance

1. Set varStatus = Approved by Clinical Director — Compliance Review
2. Continue to Step 5 (Compliance approval)
5
Approval — Compliance Team
Start and wait for an approval · Custom Responses

Only reached after Emily approves. Compliance receives the full IR PLUS Emily's notes. If compliance pushes back, the loop goes back to Emily (Step 6). Use a Do Until loop wrapping steps 5–6 so the Emily ↔ Compliance back-and-forth can repeat until both approve.

⚠️
Wrap Steps 5–6 in a Do Until loop. Condition: varStatus equals Both Approved. This handles the compliance→Emily→compliance cycle without building duplicate branches. Set a reasonable iteration limit (e.g., 5) to prevent infinite loops.
Approval type Custom Responses - Wait for one response Assigned to compliance@chollabehavioralhealth.com Response options Approve — Finalize and Return to Clinical Director Details Full IR + Emily's approval notes from varAllComments formatted as a table Subject override [ACTION REQUIRED] Compliance Review: @{varIRNumber} — Approved by Clinical Director Importance High

After this action resolves, append compliance response to varAllComments using the same pattern as Step 3.

6
Condition — Check Compliance Response (inside Do Until loop)
Condition
Condition outputs('Compliance_Approval')?['body/responses'][0]['displayText'] equals Return to Clinical Director

YES — Return to Emily

1. Set varStatus = Returned by Compliance — Re-review by Emily
2. Approval action → Emily again (same as Step 3 but include compliance notes)
3. Subject: [RE-REVIEW] Compliance Notes: @{varIRNumber}
4. Append Emily's new response to varAllComments
5. Loop continues → back to Compliance approval (Step 5)

NO — Both Approved

1. Set varStatus = Both Approved
2. Do Until exits — proceed to PDF generation (Step 7)
7
Select Notes for Final Record (Checkbox Logic)
Post Adaptive Card and wait for a response · Approvals

Before generating the PDF, send one final card to Emily (or the final approver) that shows all collected comments and lets them check which ones to include in the permanent record. The response populates varSelectedNotes.

ℹ️
Implementation approach: Use another Custom Response approval sent to Emily. In the details field, list all comments from varAllComments with a numbered reference. Ask Emily to type the numbers of comments to include (e.g., "1,2,4"). The comment field response becomes varSelectedNotes. Alternatively, include a pre-formatted "All Notes Summary" and give Emily only Approve/Exclude options — simpler and less error-prone for Outlook-only users.
Approval type Custom Responses - Wait for one response Assigned to emily.obrien@chollabehavioralhealth.com Response options Include All Notes in PDF · Include Selected Notes Only Details List all comments numbered. Instruct: "If selecting notes, type the note numbers to exclude in the comments field below." On "Include All" Set varSelectedNotes = full formatted string of varAllComments On "Selected Only" Run a Filter array action to exclude items by the numbers Emily entered in comments
